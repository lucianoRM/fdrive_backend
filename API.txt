Corriendo localmente al host, la dirección del server es: localhost:8000.

En caso de error, la respuesta siempre es { "result" : false , "errors" : ["<Error Message>"] }

ADD USER:
	url: <dirección server>/users?email=<email>&password=<password>
	Método: GET
	Body:
	Respuesta: En caso de no existir ya un usuario con ese mail se recibe { "result" : true } .


LOGIN:
	url: <dirección server>/login?email=<email>&password=<password>
	Método: GET
	Body:
	Respuesta: En caso de no existir el email, se recibe un mensaje de error.
			En caso satisfactorio se recibe un token para que el usuario pueda operar:
				{ "result" : true , "token" : "<token>" }

GET USERS:
	url: <dirección server>/login?email=<email>&token=<token>
	Método: GET
	Body:
	Respuesta: Si no hay problemas con la base de datos, siempre se supone exitoso.
			   Devuelve todos los users que hay sin contar a quien hizo la request:
			     { "result" : true , "users" : [
													{ "email" : <email>,
													  "password" : <password>,
													  "totalQuota" : <totalQuota>,
													  "freeQuota" : <freeQuota>,
													  "tokens" : [ {"token" : <token>, "expiration" : <expiration>} , ... ],
				(opcional - sólo si lo tiene)		  "name" : <name>,
				(opcional - sólo si lo tiene)		  "lastLocation" : <location>,
				(opcional - sólo si lo tiene)		  "pathToProfilePicture" : <path>
													} ,
													...
											   ]
			     }

SAVE FILE: Para NEW files (the user sending it will be the owner)
	url: <dirección server>/files
	Método: POST
	Body: {
		“email”:”<email>”,
		“token”:”<token>”,
		“name”:”<name>”,
		“extension”:”<extension>”,
		"path":"<path>",
		“tags”:[<tag1>, <tag2>, ...]
	}
	Respuesta: En caso de que todos los campos sean correctos, { "result" : true , "fileID" : <id> }.

SAVE FILE: Para OLD files (the user sending it will be creating a new version)
	url: <dirección server>/files
	Método: POST
	Body: {
		“email”:”<email>”,
		“token”:”<token>”,
		“name”:”<name>”,
		“extension”:”<extension>”,
		"id":<id>,
		“tags”:[<tag1>, <tag2>, ...]
	}
	Respuesta: En caso de que todos los campos sean correctos y el usuario tenga permisos para acceder al archivo, { "result" : true }.

LOAD FILE:
	url: <dirección server>/files?email=<email>&token=<token>&id=<id>
	Método: GET
	Body:
	Respuesta: En caso de que el token sea correcto,
	    {
    		"extension" : "<extension>",
            "id" : <id>,
            "lastModified" : "<timestamp>",    // Format: Thu Oct 22 03:24:19 2015
            "lastUser" : "<user>",
            "name" : "<file1>",
            "owner" : "<owner>",
            "tags" : [ "<tag1>", "<tag2>", ... ]
    	}

GET USER'S FILES:
    url: <dirección server>/userfiles?email=<email>&token=<token>&path=<path>
    Método: GET
    Body:
	Respuesta:
		{ "result" : true ,
		  "content" : 
				{
				  "folders" : [ "<folder1>" , ... ],
				  "files" : [ <id1> , ... ],
				  "filesNames : [ "<name+extensionFromId1>" , ... ]
				}
        }


ERASE FILE:
	url: <dirección server>/files?email=<email>&token=<token>&path=<path>&id=<id>
	Método: DELETE
	
	Respuesta: En caso de que el token sea correcto y corresponda al email, se elimina el archivo y se recibe 
	
	un mensaje de success con el id del archivo borrado.


SHARE FILE:
	url: <dirección server>/share
	Método: POST
	Body:{
		“email”:”<email>”,
		“token”:”<token>”,
		“id”:”<id>”,
		"users" : [ <email1>, <email2>, ...]
		}
	Respuesta: Si el que lo pide es el dueño y todos los usuarios a los que se les quiere compartir existen 
					y no tiene ya compartido el archivo, hay éxito y se devuelve:
				{ "result" : true }


UPLOAD FILE:
	url: <dirección server>/filesupload?email=<email>&token=<token>
	Método: POST
	Multipart
	Body:{
		"upload":<data>
	}
	Respuesta: {"result": true}

ADD FOLDER:
	url: <dirección server>/folders?email=<email>&token=<token>&path=<path>&name=<name>
	Método: POST
	Respuesta: {"result": true}

RENAME FOLDER:
	url: <dirección server>/folders?email=<email>&token=<token>&path=<path>&oldname=<oldname>&newname=<newname>
	Método: PUT
	Respuesta: {"result": true}

GET SEARCHES:
    url: <dirección server>/searches?email=<email>&token=<token>&typeofsearch=<typeofsearch>&element=<element>
    Método: GET
    Body:
	Respuesta:
		{ "result" : true ,
		  { "files" : [ {"id": <id>, "path": "<path>"} , ... ] }
		}


PROPOSED NEW FILES API

METADATA
/files/:id/metadata            [GET]    Obtener la metadata de la última versión de un archivo
/files/:id/:version/metadata   [GET]    Obtener la metadata de una versión de un archivo

/files/:id/metadata            [POST]   Subir la metadata de una versión nueva de un archivo
/files/metadata                [POST]   Subir la metadata de un archivo nuevo

/files/:id/:version/metadata   [PUT]    Cambiar la metadata de la última versión de un archivo

DATA
/files/:id/:version/data       [GET]    Obtener una versión de un archivo
/files/:id/data                [GET]    Obtener la última versión de un archivo
/files/:id/:version/data       [POST]   Subir una versión de un archivo

GENERICOS
/files/:id                     [DELETE]
